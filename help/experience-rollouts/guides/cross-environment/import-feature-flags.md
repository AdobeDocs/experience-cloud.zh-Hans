# 导入功能标志 {#import-feature-flags}

通过体验转出，可将功能标记从较低的环境（例如，暂存）导入到较高的环境（例如，生产）中。 这样可避免手动重新创建标记配置，并降低环境之间的配置漂移风险。

## 先决条件 {#prerequisites}

要使用导入工作流，您的应用程序实例必须跨环境链接。 请参阅[将环境关联到应用程序](associate-environments.md)。

## 步骤1：转到目标环境和应用程序 {#step-1}

登录到&#x200B;**目标**&#x200B;环境的控制台 — 您希望将标志&#x200B;*导入到*&#x200B;中的环境。 从“功能标志”页上的应用程序下拉列表中选择要将标志导入到的应用程序。

>[!IMPORTANT]
>
>您的当前环境和选定的应用程序必须是&#x200B;**目标** — 而不是源。 例如，要将标志从暂存导入生产环境，请登录到生产控制台并选择您的生产应用程序。

## 第2步：打开导入对话框 {#step-2}

选择&#x200B;**导入功能标志**。 此时将打开一个对话框，其中显示了根据为您的应用程序配置的链接环境预填充的源环境和应用程序。 如果需要，您可以从对话框中的下拉列表更改源环境和应用程序。

## 第3步：选择要导入的功能标志 {#step-3}

从源环境中的功能标记列表中，选择要导入的标记。 您可以一次选择一个、多个或所有标志。

## 步骤4：处理现有标志（如果需要） {#step-4}

如果目标环境中已存在具有相同键的功能标记，则体验转出将要求您确认是否覆盖它。 在确认之前查看现有标志配置，因为覆盖操作会将目标标志的设置替换为来自源的设置。

## 重要说明 {#important-notes}

在导入功能标记时，请牢记以下几点：

* 导入的标记在目标环境中始终设置为&#x200B;**OFF**&#x200B;状态，无论它们在源环境中的状态如何。 导入后，必须手动启用它们。
* 如果计划在源环境中的未来日期和时间激活标志，则该计划&#x200B;**不会**&#x200B;延续。 如果需要，您必须在目标环境中设置新计划。
