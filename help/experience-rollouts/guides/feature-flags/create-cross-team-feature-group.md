# 创建跨团队功能组 {#create-cross-team-feature-group}

## 先决条件 {#prerequisites}

在创建跨团队功能组之前，请确认以下各项：

* 您有权访问控制台 — 请参阅[登录到控制台](../console/log-in-to-the-console.md)
* 您属于某个团队，您的应用程序已载入
* 您具有&#x200B;**功能管理员**&#x200B;角色 — 请参阅[用户角色](../teams/user-roles.md)
* 您已创建要包含的功能标志 — 请参阅[创建您的第一个功能标志](create-your-first-feature-flag.md)

跨团队功能组允许将来自具有丰富受众定位的多个团队中应用程序的功能标记捆绑在一起。 与版本不同，此服务是完全自助的，对您可以创建的数量没有限制。 有关比较，请参阅[版本和跨团队功能组](releases-and-cross-team-feature-groups.md)。

## 步骤1：创建跨团队功能组 {#create}

从控制台的Releases部分开始创建过程：

1. 在控制台中导航到&#x200B;**功能和版本>版本**。
2. 选择&#x200B;**新建功能组（跨团队）**。

## 步骤2：基本详细信息 {#basic-details}

提供标题、键、描述和（可选）标记。 配置以下选项：

* **转出百分比** — 设置受众接收该功能的比例。
* **转出类型** — 设置为手动。 随着转出的进行，会逐步管理百分比。

>[!NOTE]
>
>跨团队功能组不支持A/B测试。 要运行A/B测试，请使用标准[功能组](create-a-feature-group.md) （应用程序必须位于同一组中）。

## 步骤3：受众 {#audience}

在&#x200B;**受众**&#x200B;选项卡上，启用&#x200B;**受众规则**&#x200B;并配置：

* **区段** — 选择预定义的受众区段。
* **其他筛选器** — 直接添加基于配置文件或基于上下文的标准。
* **应用程序** — 从您的团队或其他团队中选择一个或多个应用程序。

>[!IMPORTANT]
>
>添加来自其他团队的应用程序时，该团队的功能管理员有权将其应用程序的功能标记添加到此跨团队功能组中。 您无法自行添加其功能标记。

## 步骤4：功能 {#features}

在&#x200B;**功能**&#x200B;选项卡下，您会看到每个包含的应用程序的框：

* 对于您自己团队中的应用程序，您可以直接添加功能标记。
* 对于来自其他团队的应用程序，这些框呈灰显状态 — 相应团队的功能管理员必须添加其标志。

>[!IMPORTANT]
>
>一次只能通过一种方法提供功能标志。 向跨团队功能组添加标记会直接删除在标记上设置的任何受众或百分比转出。 已存在于另一个版本或功能组中的标记将不会显示。

## 步骤5：计划（可选） {#schedule}

您可以安排跨团队功能组在将来的日期和时间激活。 有关详细信息，请参阅[计划](schedule.md)。

## 管理状态 {#states}

创建跨团队功能组的团队拥有该功能组并且可以管理其状态。 所属团队的功能管理员成员可以使用状态下拉菜单在状态之间转换：

| State | 描述 |
|---|---|
| **保存** | 已保存，但未处于活动状态。 允许所有更改。 |
| **发布** | 针对配置的受众上线。 功能管理员可以继续更新受众标准和百分比转出。 |
| **完全转出** | 可供所有用户使用。 无法进一步限制此时间之后的受众。 |
| **基线** | 功能现在为缺省设置。 已关闭跨团队功能组。 |
| **中止** | 转出已停止。 没有用户从该组接收功能。 |

## 常见问题解答 {#faqs}

**跨团队功能组的数量是否存在限制？**
否。 与版本不同，此版本没有限制。 但是，当不再需要时，存档或禁用组。

**版本和跨团队功能组之间有何区别？**
有关详细比较，请参阅[版本和跨团队功能组](releases-and-cross-team-feature-groups.md)。
