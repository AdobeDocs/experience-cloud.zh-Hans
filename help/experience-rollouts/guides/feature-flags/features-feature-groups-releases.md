# 功能和功能组 {#features-feature-groups}

体验转出提供了两个用于管理功能转出的构件。 选择正确的特征取决于转出的范围和所涉及的特征数量。

## 这两个工件 {#artifacts}

**功能标志**
最原子化的单位。 控制单个应用程序的单个功能。 可以为定义的受众启用或禁用。

**功能组**
属于同一团队的功能标志集合。 允许作为一个单元跨多个应用程序管理多个标志。

## 比较 {#comparison}

| | 功能标记 | 功能组 |
|---|---|---|
| **管理受众所需的角色** | 产品版本所有者 | 产品版本所有者 |
| **应用程序** | 单身 | 多个（同一团队） |
| **富受众条件** | ✓ | ✓ |
| **转出百分比** | 与任何受众标准相结合 | 与任何受众标准相结合 |
| **A/B 测试** | ✓ | ✓ |
| **自助服务** | ✓ | ✓ |
| **审核记录** | ✓ | ✓ |

## 何时使用每个 {#when-to-use}

| 场景 | 使用 |
|---|---|
| 为一个应用程序测试或推出单个功能 | **功能标志** |
| 在同一团队中协调多个功能，同时启用 | **功能组** |
