# 功能、功能组和版本 {#features-feature-groups-releases}

体验转出提供了四个用于管理功能转出的构件。 选择正确的受众取决于转出的范围、涉及的团队数量以及您需要的受众定位功能。

## 四个工件 {#artifacts}

**功能标志**
最原子化的单位。 控制一个团队拥有的单个应用程序的单个功能。 可以为定义的受众启用或禁用。

**功能组**
属于同一团队的功能标志集合。 允许在一个组中跨多个应用程序作为一个单元管理多个标志。

**跨团队功能组**
跨多个团队和应用程序扩展功能组功能。 自助式，支持丰富的受众标准，但不支持缓存。

**版本**
专为跨多个团队和应用程序进行大规模、协调的转出而设计。 使用基于身份验证令牌的受众定位。 支持在服务器SDK上缓存。

## 比较 {#comparison}

| | 功能标记 | 功能组 | 跨团队功能组 | 版本 |
|---|---|---|---|---|
| **管理受众所需的角色** | 产品版本所有者 | 产品版本所有者 | 功能管理 | 版本管理器 |
| **应用程序** | 单身 | 多个（同一团队） | 多个（跨团队） | 多个（跨团队） |
| **团队** | 单身 | 单身 | 跨团队 | 跨团队 |
| **富受众条件** | ✓ | ✓ | ✓ | 有限 |
| **转出百分比** | 与任何受众标准相结合 | 与任何受众标准相结合 | 与任何受众标准相结合 | 与固定受众条件相结合 |
| **A/B 测试** | ✓ | ✓ | ✗ | ✗ |
| **服务器SDK中的缓存** | 仅默认打开/关闭标记 | 无缓存 | 无缓存 | 所有发行版均缓存于本地 |
| **自助服务** | ✓ | ✓ | ✓ | 需要支持请求 |
| **审核记录** | ✓ | ✓ | ✓ | ✓ |

## 何时使用每个 {#when-to-use}

| 场景 | 使用 |
|---|---|
| 为一个应用程序测试或推出单个功能 | **功能标志** |
| 在同一团队中协调多个功能，同时启用 | **功能组** |
| 协调不同团队中跨应用程序的功能，并具有丰富的定位功能 | **跨团队功能组** |
| 跨多个团队的大规模协调发布，具有SDK级别的缓存 | **版本** |
