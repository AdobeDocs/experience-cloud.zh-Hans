# 版本和跨团队功能组 {#releases-and-cross-team}

两个版本和跨团队功能组都支持跨多个团队和应用程序的协调转出。 正确的选择取决于您的缓存要求、受众定位需求以及您想要对过程有多大的控制权。

## 跨团队功能组 {#cross-team-feature-groups}

跨团队功能组允许您从不同团队拥有的应用程序捆绑功能标志，并使用丰富的受众标准管理这些功能标志。

主要特征：

* **自助服务** — 无需支持请求即可创建
* **富受众定位** — 支持完整的受众条件集（配置文件属性、上下文变量、百分比转出）
* **您可以创建的数字没有限制**
* **无缓存** — 功能标志评估需要API调用；不支持SDK级别的缓存

有关分步创建说明，请参阅[创建跨团队功能组](create-cross-team-feature-group.md)。

## 版本 {#releases}

此版本专为需要SDK级别缓存的大型协调启动设计。 发行版本使用基于身份验证令牌的受众定位。

主要特征：

* **需要支持请求** — 版本不是完全自助的；请与Experience Rollout支持人员联系以创建一个
* **SDK缓存** — 所有版本数据都缓存在服务器SDK本地，从而减少了API调用
* **有限的受众条件** — 受众规则基于身份验证令牌（国家/地区、电子邮件、用户ID、百分比等）
* **有限的活动版本** — 一次可以存在有限数量的活动版本；计划在三个月内关闭或核准版本

## 比较 {#comparison}

| | 跨团队功能组 | 版本 |
|---|---|---|
| 自助服务 | ✓ | 需要支持请求 |
| 丰富的受众条件 | ✓ | 有限 |
| A/B 测试 | ✗ | ✗ |
| SDK缓存 | ✗ | ✓ |
| 活动限制 | 无限制 | 有限 |

## 推荐 {#recommendation}

如果您的用例涉及丰富的受众定位和自助管理，请使用&#x200B;**跨团队功能组**。 如果您的后端服务需要SDK级别的缓存，并且版本更简单的受众条件已足够，请使用&#x200B;**版本**。
