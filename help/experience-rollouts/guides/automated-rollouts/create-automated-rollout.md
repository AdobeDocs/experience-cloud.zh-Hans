# 创建自动转出 {#create-automated-rollout}

自动转出允许您在一个会话中定义所有转出阶段及其计划。 体验转出会自动从一个阶段前进到下一个阶段，无需您返回控制台即可完成每个步骤。 有关概述，请参阅[自动转出概念](automated-rollout-concept.md)。

功能标记、功能组和跨团队功能组支持自动转出。

## 步骤1：启用自动转出 {#enable-automated-rollout}

创建新功能标志、功能组或跨团队功能组时，打开&#x200B;**基本详细信息**&#x200B;选项卡并找到&#x200B;**选择转出类型**&#x200B;选项。 默认为&#x200B;**手动**。 要启用自动转出计划，请选择&#x200B;**自动转出**。

>[!NOTE]
>
>当您选择自动转出时，基本详细信息中的转出百分比字段会自动设置为100%，并且变为不可编辑。 您将在“受众”选项卡中单独控制每个阶段的百分比。

## 第2步：制定转出计划 {#build-rollout-plan}

打开&#x200B;**受众**&#x200B;选项卡并启用&#x200B;**阶段**&#x200B;切换开关。 这将激活&#x200B;**添加阶段块**&#x200B;和&#x200B;**添加等待块**&#x200B;控件。

要构建转出计划，请按照您希望它们执行的顺序添加阶段和等待块：

### 添加阶段块 {#add-phase-block}

阶段块定义转出某个阶段的受众条件。 使用页面上已有的受众标准配置第一个阶段块。 对于每个后续阶段块，将预填充前一阶段的受众作为起点。

为每个阶段块配置受众：

* **百分比** — 设置此阶段中接收该功能的已定义受众的百分比。
* **受众规则** — 添加配置文件、上下文或基于区段的规则以定位特定用户。

>[!IMPORTANT]
>
>后续每个阶段都应扩展而不是替换前一阶段的受众。 在配置后续阶段时，请勿从之前的阶段中删除受众规则，因为这样可能会无意中排除已接收该功能的用户。

### 添加等待块 {#add-wait-block}

等待块定义两个相位块之间的暂停。 添加等待块后，请选择以下选项之一：

| 选项 | 描述 |
|---|---|
| **等待** | 相对持续时间 — 例如，等待2天，等待4小时。 在此持续时间过后，体验转出将进入下一阶段块。 |
| **等待** | 固定的日期和时间。 体验转出在指定的时间进入下一阶段块。 |

### 添加后续阶段块 {#add-subsequent-phases}

重复此过程以添加其他阶段块和等待块，直到配置完整转出计划。

## 第3步：计划或发布转出 {#schedule-or-publish}

转出计划完成后，请选择如何激活它：

* **计划** — 设置转出自动开始的未来日期和时间。 有关详细信息，请参阅[计划](../feature-flags/schedule.md)。
* **手动发布** — 打开功能标志或将功能组移动到&#x200B;**发布**&#x200B;状态以立即开始。

计划从第一阶段块开始执行。 您可以随时监控和编辑活动计划。 查看[监控和编辑转出计划](monitor-edit-rollout-plan.md)。
