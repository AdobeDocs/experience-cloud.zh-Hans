# 自动转出概念 {#automated-rollout-concept}

自动转出允许您一次性定义整个分阶段转出计划。 您无需手动返回到控制台来展开每个阶段的受众，而是预先配置所有阶段及其计划，然后自动体验转出通过它们。

功能标记、功能组和跨团队功能组支持自动转出。

## 工作原理 {#how-it-works}

转出计划由按顺序执行的&#x200B;**阶段块**&#x200B;和&#x200B;**等待块**&#x200B;组成：

* **阶段块** — 定义转出某个阶段的受众条件。 当计划到达某个阶段块时，该受众将成为该功能的活跃受众。
* **等待块** — 定义两个阶段块之间的暂停。 等待块可以是相对持续时间（例如，“等待3天”），也可以是固定日期和时间。

当您发布该功能或将其安排在将来日期时，计划即会开始。 体验转出会根据您配置的计划自动从一个块移动到下一个块。

## 相位块行为 {#phase-block-behavior}

每个后续阶段块均预先填充了上一阶段的受众。 这样可更轻松地构建增量扩展的转出，其中每个新阶段都包含新受众组以外的所有先前受众组。

>[!IMPORTANT]
>
>每个阶段块中的受众规则均可编辑，因此添加新受众标准时，必须确保不从上一个阶段中删除受众标准。 在转出过程中减少受众范围可能会导致用户意外地失去对功能的访问权限。

## 转出计划状态 {#rollout-plan-states}

转出计划启动后，可以处于以下三种状态之一：

| State | 描述 |
|---|---|
| **进行中** | 计划正在执行。 控制台中的进度指示器会显示哪个阶段块当前处于活动状态。 所有已完成的块均被锁定，无法编辑。 |
| **暂停** | 计划已被搁置。 上一个已完成的阶段块中的受众保持活动状态。 在暂停时，仍可以编辑未来阶段和等待块。 |
| **已中止** | 该计划已永久停止。 上一个已完成的阶段块中的受众保持活动状态。 不允许对计划进行进一步编辑。 中止计划不会关闭功能 — 如果要停止提供功能，必须单独更改功能状态。 |
