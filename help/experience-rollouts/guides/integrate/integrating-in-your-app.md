# 在应用程序中集成体验转出 {#integrate}

此部分提供按应用程序类型组织的所有Experience Rollout客户端的集成准则。

## 选择集成路径 {#choose}

选择与您的应用程序类型匹配的指南：

1. [启动指南](startup-guide.md) — 从此处开始，大致了解所有集成步骤
2. [桌面应用程序](desktop-applications.md) — 桌面应用程序的直接REST API集成
3. [移动应用程序](mobile-applications.md) — 移动应用程序的REST API集成
4. [Web应用程序](web-applications.md) — Web属性的REST API集成
5. [Web服务](web-services.md) — 后端服务的服务器端SDK集成
6. [SDK](sdks.md) — SDK架构、先决条件和可用SDK
7. [集成步骤](integration-steps.md) — 详细的分步集成说明

## 应用程序类型概览 {#overview}

| 应用程序类型 | 推荐的集成 |
|---|---|
| **Web服务/后端** | Java SDK或Node.js SDK |
| **Web应用程序** | REST API — 功能API V3 |
| **移动应用程序** | REST API — 功能API V3 |
| **桌面应用程序** | REST API — 功能API V2 |
