# 功能标志和功能组中的受众 {#audience-overview}

体验转出为功能标记和功能组提供丰富的受众定位。 您可以组合配置文件属性、上下文变量和受众区段，以精确控制哪些用户接收功能。

## 配置文件规则 {#profile-rules}

配置文件规则允许您根据属性（如国家/地区、电子邮件域、帐户类型、用户ID等）定位用户。 您可以使用AND/OR运算符组合多个属性，并为复杂的定位方案构建嵌套逻辑。

要添加配置文件规则，请转到功能标志或功能组的&#x200B;**受众**&#x200B;选项卡，并在&#x200B;**配置文件**&#x200B;下添加条件。

您可以将一组受众规则另存为可重复使用的&#x200B;**受众**，以便在多个标志和组中使用。

有关基于上下文的定位（例如，按用户的活动语言或客户端类型定位），请参阅[在受众规则中使用上下文](using-context-in-audience-rules.md)。

## 区段 {#segments}

功能标记和功能组支持受众区段。 通过区段，您可以：

* 在多个功能标志和组中使用单个预定义受众定义
* 在受众定位中包含&#x200B;**基于事件的标准** — 单独使用个人资料规则是不可能的

要使用区段，请转到&#x200B;**受众**&#x200B;选项卡，然后从列表中选择一个或多个区段。 您可以使用AND/OR运算符组合多个区段，并在这些区段旁边添加其他配置文件或上下文过滤器。

>[!NOTE]
>
>您需要&#x200B;**测试经理**&#x200B;角色或更高版本才能创建受众区段。

## 受众计算器 {#audience-calculator}

在定义受众条件后，选择底部栏中的&#x200B;**计算**&#x200B;以估计符合条件的用户数。 这有助于在激活功能之前验证定位的覆盖范围。

>[!NOTE]
>
>在标准中包含上下文变量时，受众计算器不会返回值，因为上下文值是在运行时确定的，无法提前预测。
