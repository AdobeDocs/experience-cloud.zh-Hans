# 创建新团队 {#create-a-new-team}

如果您的项目在“体验转出”中不存在团队，则需要先请求一个团队，然后才能开始创建功能标记。 团队创建是通过支持请求处理的。

## 开始之前 {#before-you-begin}

确认您的项目不存在团队。 请与您的产品或工程主管核实，或查看[团队及其管理员](teams-and-admins.md)。 如果某个团队已存在，请改为请求加入该团队 — 请参阅[请求访问](request-access.md)。

## 提交团队请求 {#submit-request}

联系Experience Rollout支持以请求新团队。 在您的请求中提供以下信息：

| 字段 | 描述 |
|---|---|
| **团队名称** | 您的团队的唯一名称。 |
| **定位模式** | 您的团队将使用的定位模式： [用户级定位或组织和环境级定位](../../getting-started/experience-rollouts-modes.md)。 |
| **环境** | 您需要的环境：暂存和/或生产。 |
| **团队管理员** | 将担任团队管理员的一个或多个人员的电子邮件地址。 |

>[!NOTE]
>
>团队管理员负责管理成员和审批访问请求。 指定至少两名管理员，以避免在一个管理员不可用时出现访问间隙。

## 创建团队后 {#after-creation}

创建团队后，管理员可以开始以自助方式添加成员。 新成员通过访问管理系统请求访问，管理员批准或拒绝这些请求。

有关成员加入团队应遵循的步骤，请参阅[请求访问权限](request-access.md)。
